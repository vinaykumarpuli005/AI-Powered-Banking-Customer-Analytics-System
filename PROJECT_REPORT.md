# Project Report

## Title

AI-Powered Banking Customer Analytics System

## Problem Statement

Banks need intelligent analytics systems to identify customers who may churn, understand customer behavior, detect suspicious transactions, and support data-driven business decisions.

## Objective

The objective of this project is to build an end-to-end AI-powered banking analytics system using Python, SQL Server, Machine Learning, Deep Learning, Streamlit, and Power BI.

## Methodology

1. Loaded the banking churn dataset.
2. Performed exploratory data analysis using Python.
3. Cleaned and prepared customer data.
4. Built a Random Forest model for churn prediction.
5. Built a K-Means clustering model for customer segmentation.
6. Built an Isolation Forest model for fraud/anomaly detection.
7. Built an Autoencoder neural network for deep learning anomaly detection.
8. Generated product recommendations using rule-based logic.
9. Imported customer data into SQL Server using SSMS.
10. Created SQL Server queries and views for analytics.
11. Exported processed data for Power BI.
12. Built six Power BI dashboards.
13. Created a Streamlit app for real-time churn prediction.

## Models Used

| Model | Type | Purpose |
|---|---|---|
| Random Forest Classifier | Supervised ML | Churn prediction |
| K-Means Clustering | Unsupervised ML | Customer segmentation |
| Isolation Forest | Anomaly Detection | Suspicious transaction detection |
| Autoencoder | Deep Learning | Transaction anomaly detection |

## Key Insights

- Customer churn varies across geography, gender, age group, and product count.
- Inactive customers and customers with low credit scores can be treated as higher risk.
- Customer segmentation helps group customers for targeted banking strategies.
- Transaction and fraud dashboards help monitor suspicious activity patterns.
- Power BI dashboards convert model outputs into business-friendly insights.
